# 迷局 AI — 项目优化分析报告

> **分析日期**: 2026-03-26  
> **分析范围**: 全部源码（18 个文件，含 CSS / JS / 剧本数据）

---

## 🐛 一、需要修复的 Bug

### 1.1 CSS 变量值格式错误

**文件**: `src/style.css` 第 27 行  
**问题**: `--color-text-muted` 值多了一个 `0`  
```css
/* ❌ 当前 */
--color-text-muted: #6868880;
/* ✅ 修正 */
--color-text-muted: #686888;
```
**影响**: 所有使用 `--color-text-muted` 的元素颜色可能渲染异常，包括滚动条 hover、NPC 侧边栏标题、生成动画中断点等。

---

### 1.2 XSS 注入风险

**文件**: `src/main.js` 第 61 行  
**问题**: API 设置弹窗中，`settings.apiKey` 等值直接拼接进 HTML，如果 localStorage 被注入恶意内容（或用户输入含引号/标签的值），可能导致 XSS。
```javascript
// ❌ 当前
value="${settings.apiKey}"
// ✅ 应转义后再拼接
```
**同类问题还存在于**:
- `src/pages/home.js` — 删除确认弹窗中的 `title` 变量
- `src/pages/custom-scenario.js` — 向导中用户输入的 `theme` / `description` 值

---

### 1.3 移动端"线索板"和"提交推理"按钮丢失

**文件**: `src/style.css` 第 1150 行  
**问题**: 移动端 `.npc-sidebar-actions { display: none; }` 直接隐藏了底部操作按钮。
```css
/* ❌ 当前：移动端完全隐藏了关键按钮 */
.npc-sidebar-actions { display: none; }
```
**影响**: 移动端用户 **无法查看线索板，也无法提交推理**，导致游戏无法进行。

---

### 1.4 游戏页 AI 模式未使用流式输出

**文件**: `src/pages/game.js` 第 251 行  
**问题**: 调用 `aiService.chat()` 时未传入 `onChunk` 回调，虽然 AI Service 支持流式输出，但游戏页实际使用的是非流式模式，需要等待完整回复后才显示。
```javascript
// 当前：非流式调用
reply = await aiService.chat(npc.systemPrompt, history, text)
// 建议：传入 onChunk 实现逐字显示
```
**影响**: 用户需要等待较长时间（尤其长回复），体验不佳。

---

### 1.5 自定义剧本工坊缺少风格/时代验证

**文件**: `src/pages/custom-scenario.js`  
**问题**: Step 2 → Step 3 跳转时不检查 `style` 和 `era` 是否已选择（两项无默认值）。如果用户未选择直接点击"开始生成"，会将空值传给 AI prompt，导致 AI 生成质量降低。

---

### 1.6 `消失的列车` 剧本 answer.suspect 逻辑可商榷

**文件**: `src/scenarios/vanishing-train.js` 第 222 行  
**问题**: `answer.suspect` 设为 `liu-xiao`（小刘），但故事真相是张文远自导自演的失踪，小刘只是帮手。严格来说"凶手"概念不完全匹配——但这是入门剧本的设计选择，属于设计层面可讨论。

## ⚡ 二、性能优化

### 2.1 大型 CSS 文件缺少分块

**问题**: `style.css` 单文件 35KB（1711 行），包含所有页面样式。随着功能增加会越来越臃肿。  
**建议**: 按模块拆分（`base.css` / `home.css` / `game.css` / `wizard.css` 等），利用 Vite 的 CSS 分块能力。

### 2.2 剧本数据全量加载

**问题**: 5 个剧本文件（合计 ~83KB）在入口即全部导入，即使用户只玩一个剧本。  
**建议**: 改为动态 `import()` 懒加载，按需加载对应剧本。

### 2.3 页面切换全量重渲染

**问题**: `reRenderGame()` 方法直接 `innerHTML = renderGame()` 全量替换 DOM，会造成不必要的重绘。NPC 切换时整个聊天区域重建。  
**建议**: 局部更新对话区域和侧边栏，避免全量 DOM 替换。

## 🛡️ 三、健壮性 & 容错

### 3.1 AI 返回 JSON 解析失败无用户提示

**文件**: `src/pages/custom-scenario.js` 第 638 行  
**问题**: `JSON.parse(cleaned)` 如果 AI 返回的不是合法 JSON（很常见），`catch` 块会抛出通用错误，但错误信息不明确。  
**建议**: 增加专门的 JSON 解析重试逻辑或让 AI 重新生成。

### 3.2 线索提取 API 调用无超时控制

**文件**: `src/game/clue-extractor.js`  
**问题**: 每次 NPC 回复后都会额外调用一次 AI 来提取线索，但没有超时控制。如果 API 响应慢，用户会感到对话延迟。  
**建议**: 增加超时机制（如 5 秒），超时后降级到关键词模式。

### 3.3 localStorage 存储溢出无处理

**问题**: 自定义剧本和游戏状态都存储在 localStorage（通常 5-10MB 限额），但没有容量检测和清理机制。  
**建议**: 增加存储容量检查，或在自定义剧本数量达到上限时提示用户。

### 3.4 页面刷新后游戏状态恢复不完整

**问题**: `gameState` 从 localStorage 恢复时，`scenario` 数据不会自动恢复（依赖 `sessionStorage` 中的 `miju-selected-scenario`）。如果用户在游戏中途关闭 Tab 页后重新打开，可能出现状态错乱。

## 📱 四、移动端适配

### 4.1 移动端功能缺失（重要）

如 1.3 所述，移动端隐藏了线索板和提交推理按钮。应该改为：
- 将按钮移到游戏页顶部 Header 或底部固定栏
- 或使用浮动 FAB 按钮

### 4.2 推理页/结果页无移动端适配

**问题**: `deduction` 和 `result` 页面在小屏幕上没有额外的响应式处理，表单元素间距和字体可能过大。

### 4.3 介绍页角色卡片溢出

**问题**: 4 个角色卡片在小屏幕上可能无法全部显示，虽然设了 `flex-wrap: wrap`，但 160px 的固定宽度在极窄屏幕上仍可能拥挤。

---

## 🎮 五、游戏体验优化

### 5.1 增加游戏存档功能

**现状**: 仅保存最后一局状态。如果开始新游戏，旧游戏进度丢失。  
**建议**: 支持多局存档，提供"继续游戏"入口。

### 5.2 AI 评分替代关键词评分

**现状**: 动机和手法评分基于硬编码的关键词匹配，灵活性低。  
**建议**: 用 AI 语义评分（0-30分），与关键词评分取更高值作为最终分数。

### 5.3 对话上下文窗口管理

**现状**: 所有对话历史都发送给 AI（`history.map(...)` 无截断），对话过多时可能超出 Token 限制导致 API 报错。  
**建议**: 限制发送给 AI 的历史消息数量（如最近 20 条），或做 Token 计数截断。

### 5.4 NPC 回复后自动滚动不一致

**现状**: `scrollChatToBottom()` 在 `requestAnimationFrame` 中执行，但 `appendMessage` 是在 DOM 插入后立即调用，可能存在时序问题。

### 5.5 回合用完自动跳转的体验

**现状**: 回合用完后 1 秒自动 `reRenderGame`，只是刷新页面并显示提示。  
**建议**: 弹窗提示"回合已用完"并提供"提交推理"按钮，更明确。

## 🔒 六、安全 & 隐私

| 问题 | 说明 | 严重程度 |
|------|------|----------|
| API Key 明文 localStorage | 可被浏览器扩展或 XSS 窃取 | ⚠️ 中 |

| HTML 注入 | 多处 `innerHTML` 拼接未转义用户输入 | ⚠️ 中 |
