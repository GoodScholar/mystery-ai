# 迷局 AI — 产品需求文档（PRD）

> **版本**: v1.0.0  
> **更新日期**: 2026-03-26  
> **产品定位**: 沉浸式 AI 剧本杀推理游戏（Web 端）

---

## 一、产品概述

### 1.1 产品简介

「迷局 AI」是一款基于大语言模型（LLM）的 **单人沉浸式剧本杀推理游戏**。玩家扮演侦探角色，通过与 AI 驱动的 NPC 自由对话来收集线索、推理真相，最终提交自己的推理判断并获得评分。

### 1.2 核心价值

| 维度 | 说明 |
|------|------|
| **沉浸感** | AI NPC 具有独立人格和秘密，对话自由度高，非固定选项 |
| **可玩性** | 5 个预设剧本 + AI 自动生成无限新剧本 |
| **策略性** | 有限对话回合机制，迫使玩家高效提问 |
| **低门槛** | 纯 Web 端，无需安装，打开即玩 |

### 1.3 技术栈

| 层级 | 技术选型 |
|------|----------|
| 构建工具 | Vite 5.x |
| 前端框架 | 原生 JavaScript（无框架） |
| 路由方案 | 自研 Hash-based SPA Router |
| 样式方案 | 原生 CSS（Google Fonts: Noto Sans SC / Playfair Display） |
| AI 服务 | OpenAI 兼容 API（默认 DeepSeek，可切换 Gemini 等） |
| 状态持久化 | localStorage |
| 部署方式 | Vite 静态构建 |

## 二、用户角色与使用场景

### 2.1 目标用户

- 推理爱好者 / 剧本杀玩家
- 对 AI 交互游戏感兴趣的泛娱乐用户
- 想独自体验推理乐趣的单人玩家

### 2.2 核心使用场景

1. 选择一个预设剧本，与 AI NPC 对话收集线索并推理破案
2. 输入主题关键词，让 AI 自动生成一个全新的剧本并立即游玩
3. 管理自定义剧本（预览、删除）

### 3.2 模块详细说明

#### 3.2.1 首页 — 剧本大厅

| 功能项 | 说明 |
|--------|------|
| 剧本卡片展示 | 以网格布局展示所有剧本（预设 + 自定义），包含封面渐变、Emoji 图标、标题、简介、难度/人数/时长标签 |
| 创建新剧本 | 独立入口卡片，跳转至自定义剧本工坊 |
| 删除自定义剧本 | 自定义剧本卡片右上角显示删除按钮，含二次确认弹窗 |
| API 设置 | 底部"API 设置"按钮，弹窗配置 API Key / Base URL / 模型名称 |

#### 3.2.2 剧本介绍页

| 功能项 | 说明 |
|--------|------|
| 打字机效果 | 背景故事文字逐字显示，沉浸感 |
| 角色卡片 | 展示所有 NPC 的 Emoji、姓名、身份、状态 |
| 规则提示 | 提示玩家可用对话回合数 |
| 开始调查 | 初始化游戏状态，进入游戏页 |

#### 3.2.3 游戏核心页

**NPC 侧边栏**
- 展示所有嫌疑人，含头像、姓名、身份
- 显示已与每位 NPC 的对话次数
- 点击切换当前对话 NPC
- 底部"线索板"和"提交推理"按钮

**对话区域**
- 实时显示与当前 NPC 的对话历史
- 支持文字输入，Enter 发送，Shift+Enter 换行
- 输入框自动高度调整（最高 120px）
- NPC 回复前显示打字指示器（三个跳动圆点）
- 支持 AI 模式（真实对话）和 Mock 模式（预设回复轮播）

**线索系统**
- 侧滑线索面板，展示已收集线索
- 每条线索包含：来源 NPC、标题、内容
- 发现新线索时浮动通知提示（3秒自动消失）
- 线索提取支持两种模式：
  - **AI 模式**: 将对话内容发送给 LLM 分析是否触发线索
  - **关键词模式**: 基于 NPC 回复中的关键词匹配（需匹配 ≥2 个关键词）

**回合机制**
- 顶部显示剩余对话次数（每次玩家发言消耗 1 次）
- 剩余 ≤5 次时高亮脉冲警告
- 用完后禁止输入，提示提交推理

#### 3.2.4 推理提交页

| 功能项 | 说明 | 分值 |
|--------|------|------|
| 选择凶手 | 点选嫌疑人卡片（单选） | 40 分 |
| 作案动机 | 自由文字描述 | 30 分 |
| 作案手法 | 自由文字描述 | 30 分 |
| 线索参考 | 展示已收集的线索供参考 | — |
| 效率加分 | 剩余回合数 × 2（上限 10 分） | 0-10 分 |

**评分算法**
- 凶手判断：完全匹配 → 40 分，否则 0 分
- 动机/手法：基于关键词匹配，匹配数 / 3 × 满分，上限 30 分
- 三项均需填写才可提交

#### 3.2.5 结果页

| 功能项 | 说明 |
|--------|------|
| 分数动画 | 数字从 0 递增至最终分数 |
| 等级评定 | S(≥95) / A(≥80) / B(≥60) / C(≥40) / D(<40)，每级有对应标签 |
| 分数明细 | 凶手/动机/手法/效率/线索数/对话数 |
| 真相揭秘 | 展示完整真相故事（支持 Markdown 加粗和标题） |
| 再玩一次 | 重置当前剧本的游戏状态重新开始 |
| 返回大厅 | 完全重置，回到首页 |

#### 3.2.6 自定义剧本工坊（4 步向导）

**Step 1 — 灵感激发**
- 12 个主题模板（校园怪谈、孤岛惊魂、太空密室等）
- 点选模板自动填入主题名称
- 也可自由输入主题名
- 可选：补充描述（核心诡计、角色类型、剧情走向）

**Step 2 — 剧本设定**
- 故事风格：本格推理 / 变格惊悚 / 轻松搞笑 / 古风文艺 / 硬核烧脑
- 时代背景：现代都市 / 民国旧影 / 古代王朝 / 遥远未来
- 难度等级：入门(⭐⭐) / 进阶(⭐⭐⭐) / 困难(⭐⭐⭐⭐) / 大师(⭐⭐⭐⭐⭐)
- NPC 数量：3 人 / 4 人
- 可选：特殊要求

**Step 3 — AI 生成**
- 动画播报 4 个阶段（构思诡计 → 塑造人物 → 埋设线索 → 完善真相）
- 实际调用 LLM 生成完整 JSON 剧本数据
- 失败时提供重试按钮

**Step 4 — 预览与微调**
- 展示生成的剧本卡片（封面、标题、简介、角色、线索数）
- 标题和简介支持点击编辑
- 可返回重新设定或重新生成
- 确认后保存至 localStorage 并跳转游戏

## 四、数据模型

### 4.1 剧本数据结构

```javascript
{
  id: String,              // 剧本唯一 ID
  title: String,           // 剧本标题
  emoji: String,           // 主题 Emoji

  maxRounds: Number,       // 最大对话回合数

  },
  brief: String,           // 一句话简介

    id: String,            // 线索 ID
    source: String,        // 来源 NPC ID
```

## 六、AI 服务设计

### 6.1 API 配置

- **优先级**: localStorage 配置 > `.env` 环境变量 > 默认值
- **默认配置**: DeepSeek API（`deepseek-chat` 模型）
- **兼容性**: 任何 OpenAI 兼容格式的 API（Gemini、GPT 等）
- **配置项**: API Key / Base URL / 模型名称

### 6.2 对话服务

| 参数 | 值 |
|------|-----|
| 最大输出 Token | 500 |
| Temperature | 0.8 |
| 流式输出 | 支持（可选） |
| 重试机制 | 429 错误自动重试 3 次（指数退避 2s/4s/8s） |
| 错误处理 | 网络失败、API 错误、解析错误均有提示 |

### 6.3 线索提取

- AI 模式：将最新对话发送给 LLM，返回 JSON 判断是否触发线索
- 降级模式：AI 失败时自动降级为关键词匹配
- Mock 模式：无 API Key 时纯关键词匹配

### 6.4 剧本生成

- 一次 LLM 调用生成完整剧本 JSON（含角色、线索、答案、真相故事）
- 包含详细的 system prompt 模板确保输出质量
- 支持解析失败后重试

## 十二、当前已知限制

| 限制项 | 说明 |
|--------|------|
| 单人模式 | 仅支持 1 名玩家 |
| 无后端 | 所有数据存储在浏览器 localStorage，清除即丢失 |
| 评分方式 | 动机/手法评分基于关键词匹配，非 AI 语义评估 |
| 自定义剧本质量 | 依赖 LLM 输出质量，可能出现逻辑不一致 |
| 无游戏存档 | 仅保存最近一局游戏状态 |
| 不支持流式回复渲染 | 游戏页对话使用非流式模式，等待完整回复后显示 |
| API Key 明文存储 | localStorage 中明文保存 API Key |

## 十三、潜在优化方向

| 方向 | 说明 |
|------|------|
| 流式回复渲染 | 游戏对话区域支持流式打字效果 |
| AI 语义评分 | 使用 LLM 评估动机/手法描述的准确性 |
| 多剧本存档 | 支持多局游戏进度保存与恢复 |
| 剧本分享 | 导出/导入自定义剧本（JSON / 链接） |
| 移动端适配 | 优化移动端触摸交互和布局 |
| 语音交互 | 语音输入 + TTS 语音回复 |
| 社区剧本 | 在线剧本广场，分享/下载社区创建的剧本 |
| 成就系统 | 根据游戏表现解锁成就徽章 |
| 多语言 | 支持英文等多语言界面 |
