# 迷局AI — 自定义剧本创作工坊 PRD v1.0

> **版本**：v1.0  |  **日期**：2026-03-26  |  **优先级**：P0  |  **预估工期**：1-2 天

---

## 1. 概述

### 1.1 背景

当前「创建新剧本」为单页简易表单，用户体验存在以下痛点：

| 问题 | 现状 | 影响 |
|------|------|------|
| 缺乏灵感 | 空白输入框，无引导 | 用户不知道填什么，放弃率高 |
| 配置单调 | 仅有主题、难度、NPC数量 3 项 | AI 缺少上下文，生成质量不稳定 |
| 等待焦虑 | 单一 loading 动画 | AI 生成需 10-20s，用户易认为卡死 |
| 无法验收 | 生成后直接进入游戏 | 不满意时只能重新走一遍全流程 |
| 缺乏管理 | 无删除/编辑入口 | 自定义剧本堆积无法清理 |

### 1.2 目标

将自定义剧本功能从「简易表单」升级为「**沉浸式分步创作工坊**」。
- 🎯 **降低创作门槛**：提供灵感模板，让零基础玩家也能秒出好剧本
- 🎯 **提升生成质量**：更多维度的创作参数 → 更精准的 AI 提示词
- 🎯 **优化等待体验**：分阶段进度播报替代单调 loading
- 🎯 **引入验收机制**：预览 → 微调 → 确认，而非一锤子买卖
- 🎯 **完善生命周期**：支持自定义剧本的删除管理

## 2. 用户故事

| # | 角色 | 行为 | 期望 |
|---|------|------|------|
| US-1 | 新手玩家 | 点击「创建新剧本」 | 看到丰富的主题灵感卡片，点一下就能开始创作 |

| US-5 | 整理控玩家 | 首页有太多自定义剧本 | 可以长按/点击删除按钮清理不需要的剧本 |

## 3. 功能设计

### 3.1 分步向导流程（Creation Wizard）

将单页表单拆分为 **4 步向导**，顶部带进度条（Stepper），每步有独立面板：

```
[ Step 1 · 灵感 ] → [ Step 2 · 设定 ] → [ Step 3 · 生成 ] → [ Step 4 · 预览 ]
```

**各步骤概览**：

| 步骤 | 名称 | 核心交互 | 可返回 |
|------|------|---------|--------|
| Step 1 | 🎭 灵感激发 | 选择或输入主题 | — |
| Step 2 | ⚙️ 剧本设定 | 选风格/时代/难度/NPC/特殊要求 | ✅ 可回 Step 1 |
| Step 3 | ✨ 虚空酝酿 | 观看生成进度播报（不可操作） | ❌ |
| Step 4 | 📋 剧本预览 | 审阅 / 编辑标题 / 重新生成 / 开始游玩 | ✅ 可回 Step 2 |

---

| 🎪 | 午夜马戏团 | 马戏团、魔术、消失 |
| 🏥 | 废弃医院 | 医院、病历、夜班 |
| 🎬 | 惊悚片场 | 片场、导演、道具 |

### 3.3 Step 2 — 剧本设定

采用分组卡片式布局，每组一个小标题：

#### 故事风格（单选）

| 选项 | 描述 |
|------|------|
| 🔍 本格推理 | 严密逻辑推理，公平解谜 |
| 🔪 变格惊悚 | 心理恐怖，氛围压迫 |
| 😂 轻松搞笑 | 喜剧元素，反转搞笑 |
| 🌸 古风文艺 | 诗词意境，情感纠葛 |
| ⚡ 硬核烧脑 | 多重反转，极限推理 |

#### 时代背景（单选）

| 选项 | 描述 |
|------|------|
| 🏙️ 现代都市 | 当下城市社会背景 |
| 🏮 民国旧影 | 20世纪初旧中国 |
| 🏯 古代王朝 | 古装宫廷或江湖 |
| 🛸 遥远未来 | 科幻或后现代世界 |

#### 基础配置

- **难度**：⭐⭐ 入门 / ⭐⭐⭐ 进阶 / ⭐⭐⭐⭐ 困难 / ⭐⭐⭐⭐⭐ 大师
- **NPC 数量**：3 人 / 4 人
- **特殊要求**（选填文本框）：如 "需要有密室诡计" "希望凶手是最不可能的人"

---

### 3.4 Step 3 — 虚空酝酿（AI 生成动画）

当 AI 请求发出后，用**阶段式播报 + 动画**替代单调 loading：

| 阶段 | 延迟 | 播报文案 | 图标动画 |
|------|------|---------|---------|
| 1 | 0s | "正在构思核心诡计..." | 🧩 拼图旋转 |
| 2 | 3s | "正在塑造嫌疑人画像..." | 👥 人影浮现 |
| 3 | 6s | "正在埋设逻辑冲突与线索..." | 🔗 链条连接 |
| 4 | 9s | "正在完善最终真相..." | ✨ 星光聚合 |

- 使用 `setInterval` 定时切换文案，与实际 AI 返回无关
- AI 响应到达后立即跳转 Step 4

#### 生成失败处理

- 显示错误提示 + 「重试」按钮
- 重试按钮带着同样的配置重新发起请求

---

AI 返回后，展示剧本卡片式预览：

│  [渐变封面 + 大 Emoji]              │

├─────────────────────────────────────┤
│  👥 角色阵容                        │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐│
│  │ 😈   │ │ 👩‍🔬 │ │ 👴   │ │ 💃   ││
│  │ 名字 │ │ 名字 │ │ 名字 │ │ 名字 ││
│  │ 身份 │ │ 身份 │ │ 身份 │ │ 身份 ││
│  └──────┘ └──────┘ └──────┘ └──────┘│
├─────────────────────────────────────┤
│  🔍 线索数量: 8 条                  │
├─────────────────────────────────────┤

**交互逻辑**：
- **标题 / 简介**：点击变为可编辑输入框
- **🔄 重新生成**：丢弃当前数据，回到 Step 3 重新请求（保留 Step 1&2 配置）
- **🎮 开始游玩**：调用 `addCustomScenario()` 写入存储 → 跳转 `/intro`

### 3.6 自定义剧本管理

**首页（`home.js`）改动**：

- 仅 `isCustom: true` 的剧本卡片右上角显示半透明 🗑️ 删除按钮
- 默认隐藏，hover 时淡入（移动端始终可见）

- 步骤切换：fadeIn + slideX（向右前进，向左后退）

- 选项胶囊选中：border-color 渐变 + 内发光
- 生成播报：文字淡入 + emoji 微动画

## 5. 技术实现要点

### 5.1 状态管理

```javascript
// custom-scenario.js 内部状态
const wizardState = {
  currentStep: 1,          // 当前步骤 1-4
  theme: '',               // 主题文本
  description: '',         // 补充描述
  style: '',               // 故事风格
  era: '',                 // 时代背景  
  difficulty: 3,           // 难度
  npcCount: 4,             // NPC 数量
  specialRequest: '',      // 特殊要求
  generatedScenario: null  // AI 生成的完整剧本对象
}
```

### 5.2 Prompt 拼装策略

将 Step 1 + Step 2 的所有参数拼装进 AI 提示词：

```
主题: ${theme}
故事风格: ${style}
时代背景: ${era}
难度: ${difficulty}
NPC数量: ${npcCount}
特殊要求: ${specialRequest}
补充描述: ${description}
```

### 5.3 涉及文件

| 文件 | 操作 | 变更内容 |
|------|------|---------|
| `src/pages/custom-scenario.js` | 重写 | 分步向导 + 预览编辑逻辑 |
| `src/pages/home.js` | 修改 | 自定义剧本卡片增加删除按钮 |
