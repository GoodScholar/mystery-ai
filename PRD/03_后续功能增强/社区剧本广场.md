# 迷局AI — 社区剧本广场 PRD v2.0

> **版本**：v2.0  |  **日期**：2026-03-26  |  **优先级**：P1  |  **预估工期**：3-5 天  
> **前置依赖**：自定义剧本创作工坊（v1.0 已完成）

---

## 1. 概述

### 1.1 背景

v1.0 完成了「自定义剧本创作工坊」，玩家可以通过 AI 生成个性化剧本。但当前存在以下问题：

| 问题 | 现状 | 影响 |
|------|------|------|
| 创意孤岛 | 自定义剧本仅存于本地 | 精心创作的好剧本无人体验，缺乏成就感 |
| 内容匮乏 | 仅 5 个预设剧本 | 玩完即走，复玩率低 |
| 缺乏社交 | 纯单机体验 | 无法了解其他玩家的评价和推荐 |
| 质量参差 | AI 生成质量不稳定 | 无法区分高质量和低质量剧本 |

### 1.2 目标

构建「**社区剧本广场**」，让玩家的自定义剧本能分享、浏览、收藏和评价：

- 🎯 **激活创作生态**：让好剧本被更多人发现和体验
- 🎯 **丰富内容池**：从 5 个预设扩展到无限社区剧本
- 🎯 **建立质量筛选**：通过评分和排行自然淘汰低质内容
- 🎯 **增强用户粘性**：社交互动 → 创作激励 → 持续回访

## 2. 用户故事

| # | 角色 | 行为 | 期望 |
|---|------|------|------|
| US-1 | 创作者 | 完成一个自定义剧本后想分享 | 一键发布到社区广场，获得唯一分享链接 |
| US-2 | 探索者 | 打开社区广场 | 看到热门/最新/精选剧本列表，快速找到感兴趣的 |
| US-3 | 挑剔玩家 | 玩完社区剧本 | 可以给剧本打分和留言评价 |
| US-4 | 收藏党 | 看到有趣的剧本但暂时没空 | 收藏到本地，随时开玩 |
| US-5 | 创作者 | 发布后想看反馈 | 在「我的发布」页查看下载量和评分 |
| US-6 | 搜索者 | 想玩特定主题的剧本 | 通过关键词搜索或标签筛选快速定位 |

## 3. 功能设计

### 3.1 整体架构

```
首页（home.js）
  ├── 预设剧本卡片
  ├── 自定义剧本卡片（本地）
  ├── [NEW] 🌐 社区广场入口
  └── ✨ 创建新剧本

[NEW] 社区广场页（/community）
  ├── 搜索栏 + 标签筛选
  ├── 排序切换（热门 / 最新 / 评分最高）
  ├── 剧本卡片列表（分页加载）
  └── 剧本详情弹窗（预览 + 收藏 + 开始游玩）

[NEW] 我的社区（/my-community）
  ├── 我发布的剧本
  ├── 我收藏的剧本
  └── 我的评价记录
```

### 3.2 社区广场页（`/community`）

#### 3.2.1 搜索与筛选

```
┌───────────────────────────────────────────┐
│  🔍 搜索剧本...                    [筛选] │
├───────────────────────────────────────────┤
│  🔥 热门   ⏰ 最新   ⭐ 高分   📌 精选    │
├───────────────────────────────────────────┤
│  标签: [推理] [恐怖] [校园] [科幻] [古风]  │
│        [搞笑] [烧脑] [密室] [全部]         │
```

- **搜索**：模糊匹配剧本标题、简介、标签
- **排序模式**：

  - ⏰ 最新 — 按发布时间倒序
  - ⭐ 高分 — 按综合评分排序
  - 📌 精选 — 人工/算法推荐（预设标记）
- **标签筛选**：点击标签切换过滤，支持多选

### 3.3 剧本发布流程

在创作工坊 Step 4（预览页）新增「📤 发布到社区」按钮：

```
Step 4 预览页
  └── [🔄 重新生成]  [📤 发布到社区]  [🎮 开始游玩]
```

#### 发布弹窗

```
┌─────────────────────────────────┐
│  📤 发布到社区广场                │
│                                  │
│  创作者昵称                       │
│  [____________]                  │
│                                  │
│  ⚠️ 发布后其他玩家可以浏览        │
│     和体验你的剧本                │
│                                  │
│  [取消]      [确认发布]           │
└─────────────────────────────────┘
```

- **创作者昵称**：首次发布时输入，之后自动记住（存 localStorage）
- 发布后生成唯一 ID，剧本标记为 `isPublished: true`
- 发布的剧本数据中**不包含** `answer` 和 `truthReveal`（防剧透），仅在用户选择游玩时完整加载

## 4. 数据存储方案

### 4.1 本地存储（localStorage）

由于迷局 AI 当前是纯前端应用，社区功能采用 **localStorage + JSON 导入导出** 的轻量方案：

```javascript
// 存储 Key 规划
const STORAGE_KEYS = {
  customScenarios: 'miju-custom-scenarios',     // 已有
  communityIndex: 'miju-community-index',        // 社区剧本索引
  communityData: 'miju-community-data-{id}',     // 单个社区剧本数据
  favorites: 'miju-favorites',                    // 收藏列表
  myPublished: 'miju-my-published',               // 我发布的
  myReviews: 'miju-my-reviews',                   // 我的评价
  userProfile: 'miju-user-profile',                // 用户昵称等
}
```

### 4.2 社区数据交换

#### 方案 A：内置预设社区剧本（MVP 推荐 ✅）

- 将精选社区剧本打包为 JS 模块（类似现有的 `midnight-gallery.js`）
- 每次版本更新时加入新的社区剧本
- 模拟评分和评价数据

优势：零后端依赖，与当前架构完全一致

#### 方案 B：JSON 导入导出

- 支持剧本导出为 `.miju.json` 文件
- 支持从文件导入剧本
- 玩家可通过 IM / 邮件 / 网盘分享文件

优势：实现真正的玩家间分享

#### 方案 C：轻量后端（后续迭代）

- 使用 Supabase / Firebase 等 BaaS 服务
- 实现实时的剧本发布、浏览、评分
- 需要用户注册登录
