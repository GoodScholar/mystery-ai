# 迷局AI — 未来优化与新增功能方向 PRD

> **文档版本**: v1.0  
> **创建日期**: 2026-03-27  
> **状态**: 已确认

---

## 一、文档概述

本文档基于迷局AI当前版本（首页 → 剧本介绍 → 游戏对话 → 结案推理 → 结果页 + 自定义剧本），梳理了未来可扩展的优化与功能方向。所有条目按**纯前端**与**需前后端联调**两大维度分类，便于团队按优先级渐进实施。

---

## 二、纯前端优化与功能增强

> 无需额外服务端 API，主要利用本地存储与浏览器端能力即可完成。

### 2.1 视听与沉浸感体验提升 (UI/UX)

| 编号 | 功能点 | 说明 | 优先级 |
|------|--------|------|--------|
| F-01 | 全局深色/浅色模式切换 | 支持 Dark/Light Mode 自由切换，配合悬疑氛围 | 🟡 中 |
| F-02 | 页面路由转场动效增强 | 淡入淡出、滑入滑出等更平滑的路由切换动画 | 🟢 低 |
| F-03 | NPC 回复打字机音效 | 为 AI 回复的打字机效果增加逼真音效与气泡状态 | 🟢 低 |
| F-04 | 环境音效与 BGM 系统 | 不同剧本绑定不同 BGM 与局部音效，纯前端 Audio API 实现 | 🟡 中 |
| F-05 | 超长对话虚拟列表 | 在 `game.js` 中引入 Virtual List，防止大量消息导致页面卡顿 | 🔴 高 |

### 2.2 玩家便捷性与本地工具

| 编号 | 功能点 | 说明 | 优先级 |
|------|--------|------|--------|
| F-06 | 存档导入/导出 (JSON) | 将 LocalStorage 进度和自定义剧本导出为 JSON，支持好友间互传 | 🟡 中 |
| F-07 | 图文线索背包 | 游戏侧边栏/抽屉自动收集聊天关键信息与线索，方便随时查阅 | 🟡 中 |
| F-08 | 全局字号与阅读模式 | 支持 UI 字体大小调节，适合不同设备和偏好的用户 | 🟢 低 |

### 2.3 系统级前端改造

| 编号 | 功能点 | 说明 | 优先级 |
|------|--------|------|--------|
| F-09 | PWA 渐进式 Web 应用 | Service Worker 缓存静态资源，支持离线加载和添加到桌面 | 🔴 高 |
| F-10 | 自定义剧本实时预览 | 编写剧本设定时分屏显示玩家视角渲染效果 | 🟡 中 |
| F-11 | 自定义剧本本地草稿箱 | 防止刷新导致未保存的剧本配置丢失 | 🔴 高 |

## 三、需要前后端联调的深度扩展

> 涉及云端存储、AI 多模态、实时通信等复杂业务逻辑，需要服务端 API 支持。

### 3.1 多模态与 AI 深层联动

| 编号 | 功能点 | 说明 | 依赖技术 | 优先级 |
|------|--------|------|----------|--------|
| B-01 | 语音交互 (TTS & ASR) | 玩家录音发指令，后端将 AI 回复转情感语音播报 | 云端 TTS/ASR API | 🟡 中 |
| B-02 | AI 动态生成图片线索 | 根据剧情进展实时生成凶案现场或物品图像 | 文生图 API (DALL-E 等) | 🟢 低 |
| B-03 | AI 辅助"一键生本" | 输入关键词自动生成完整剧本世界观、角色和核心诡计 | 大模型 API | 🔴 高 |

### 3.2 用户生态与社区 (UGC)

| 编号 | 功能点 | 说明 | 依赖技术 | 优先级 |
|------|--------|------|----------|--------|
| B-04 | 账号注册与云同步 | 微信/手机号/邮箱登录，跨设备云存档、成就系统与徽章 | 用户服务 + 云数据库 | 🔴 高 |
| B-05 | 社区剧本广场 | 高质量原创剧本一键发布，支持浏览、打分、评论和下载 | 内容服务 + CDN | 🔴 高 |
| B-06 | 创作者数据看板 | 展示自创剧本的游玩次数、通关率、平均推理时长等统计 | 数据分析服务 | 🟡 中 |

### 3.3 多人社交互动机制

| 编号 | 功能点 | 说明 | 依赖技术 | 优先级 |
|------|--------|------|----------|--------|
| B-07 | 多人联机房 | 多名玩家分别扮演不同角色，在房间内与 DM(AI) 及彼此实时交互 | WebSocket 实时通信 | 🟡 中 |
| B-08 | 全局实时线索墙 | 后端从聊天中抽取人物/时间/物品关系，前端渲染动态逻辑拓扑图 | NLP + D3.js/Echarts | 🟢 低 |

## 四、实施建议

### 推荐优先级路线

```
阶段一 (纯前端快速见效)
  → F-05 虚拟列表 → F-09 PWA → F-11 草稿箱 → F-01 深色模式

阶段二 (纯前端体验增强)
  → F-06 存档导入导出 → F-07 线索背包 → F-10 实时预览 → F-04 BGM

阶段三 (前后端联调 - 核心价值)
  → B-03 一键生本 → B-04 账号云同步 → B-05 社区广场

阶段四 (前后端联调 - 差异化)
  → B-01 语音交互 → B-07 多人联机 → B-06 数据看板 → B-08 线索墙 → B-02 图片线索
```

## 五、备注

- 所有纯前端功能均兼容现有 Vite + 原生 JS 技术栈，无框架迁移成本。
- 前后端联调功能若最终走小程序路线（参考 `PRD/04_平台转型`），需同步评估对应云开发能力。
- 各功能点的详细设计文档将在正式立项后独立编写。
